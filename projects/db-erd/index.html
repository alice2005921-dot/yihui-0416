ERD + SQLite：簽核領用與庫存

Overview

我想展示什麼

用 ERD 表達資料關聯與正規化思路，並提供可執行的 SQLite schema 與報表查詢，讓「設計」有可驗證的輸出。

設計重點

  • 單據/明細分離（支持多品項）
  • 簽核紀錄獨立（可追溯）
  • 庫存異動事件表（不可改，只能反向修正）

作品檔案

  • README
  • ERD 說明（原始 md）
  • schema.sql
  • queries.sql
看 schema 看報表查詢

如何執行

sqlite3 demo.db < schema.sql
sqlite3 demo.db < queries.sql

我已在本機測試可正常輸出：申請總覽、庫存、排行、核准耗時。
