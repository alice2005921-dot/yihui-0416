# 內部簽核 + 庫存領用系統｜需求規格（可展示作品）

## 1. 背景與目標
公司目前使用紙本/口頭方式領用耗材與設備，容易出現：
- 權責不清：誰核准、誰領用、誰出庫不一致
- 庫存不準：盤點才發現缺料/多料
- 追溯困難：找不到領用原因與使用者

**目標**：
- 用「簽核流程」確保權責清楚
- 用「庫存異動」確保資料一致
- 用「報表」讓管理可視化

## 2. 角色與權限
| 角色 | 權限 |
|---|---|
| 申請人（員工） | 建立領用申請、查看自己申請狀態 |
| 主管（核准者） | 核准/退回申請、查看部門申請 |
| 倉管（出庫者） | 根據核准單出庫、調整庫存、盤點 |
| 系統管理員 | 維護品項、使用者/部門、權限設定 |

權限原則：
- 主管只能核准「自己部門」申請
- 倉管只可處理「已核准」且「未出庫」的申請
- 出庫後不可直接修改申請內容，只能走「作廢/更正」流程

## 3. 核心流程（To-Be）

### 3.1 領用申請流程
1. 申請人建立申請：選品項、數量、用途、期望領用日
2. 系統檢查：
   - 是否超過可領用上限（可設定）
   - 庫存是否足夠（不足時提示「可預約/改品項」）
3. 送出後狀態變更為 `PENDING_APPROVAL`
4. 主管核准/退回
   - 核准 → `APPROVED`
   - 退回 → `REJECTED`（需填寫原因）
5. 倉管出庫：
   - 出庫成功 → `FULFILLED`（同時寫入庫存異動）
   - 若出庫時庫存不足 → 顯示錯誤，禁止出庫

### 3.2 作廢/更正流程（避免黑箱）
- 申請尚未出庫：可由申請人「取消」→ `CANCELLED`
- 已出庫：不可直接刪除，只能走「更正單」：
  - 建立更正單：指定原出庫單、填寫原因、輸入更正數量
  - 主管核准後，倉管執行「反向異動」修正庫存

## 4. 資料結構（概念）

### 4.1 主檔
- `Product(品項)`：sku、名稱、分類、單位、是否啟用
- `User(使用者)`：姓名、email、部門、角色
- `Department(部門)`：部門名稱

### 4.2 單據
- `IssueRequest(領用申請)`：
  - request_id, requester_id, dept_id
  - status（PENDING_APPROVAL/APPROVED/REJECTED/FULFILLED/CANCELLED）
  - reason（用途）、needed_date、created_at
- `IssueRequestItem(申請明細)`：
  - request_id, product_id, qty
- `ApprovalLog(簽核紀錄)`：
  - request_id, approver_id, action（APPROVE/REJECT）, comment, acted_at
- `StockMove(庫存異動)`：
  - product_id, move_type（IN/OUT/ADJUST/REVERSE）, qty, ref, moved_at

一致性規則：
- 只有 `IssueRequest.status = APPROVED` 的單才可產生 `StockMove(OUT)`
- `StockMove` 一旦寫入不可修改，只能新增「反向更正」move

## 5. 介面需求（頁面）
- **申請人**
  - 新增申請（品項搜尋、數量、用途、日期）
  - 我的申請列表（狀態、建立時間、主管意見）
- **主管**
  - 待核准列表（篩選：部門、日期、狀態）
  - 申請詳情（明細、用途、庫存提示）
- **倉管**
  - 待出庫列表（已核准且未出庫）
  - 出庫作業（掃描/輸入 sku、扣庫存、列印單據）
- **報表**
  - 低庫存提醒（<= 門檻）
  - 領用排行（Top 品項/部門）
  - 申請到出庫平均時間（效率指標）

## 6. 例外情境（必須處理）
- 庫存不足仍送出申請（允許，但顯示提示；主管可退回）
- 同一申請被重複出庫（系統禁止；使用 transaction/鎖）
- 主管不在（指定代理人/自動轉派）
- 品項停用（不可新增申請，但歷史資料仍可查）

## 7. 驗收標準（Acceptance Criteria）
- 建立申請後，主管可看到待核准列表，核准/退回結果會回寫到申請人列表
- 被核准的申請才可出庫，出庫成功後庫存會正確扣減
- 報表可正確統計：
  - 每月領用量
  - 低庫存清單
  - 各部門領用排行
- 對任一筆出庫紀錄，都可追溯到：
  - 申請單、簽核紀錄、出庫人、出庫時間

## 8. 測試清單（可直接拿去測）
- 建立申請：正常/庫存不足/超量/品項停用
- 簽核：核准/退回/非本部門主管不可操作
- 出庫：已核准可出庫/未核准不可出庫/重複出庫不可
- 更正：出庫後更正單流程可回沖庫存
