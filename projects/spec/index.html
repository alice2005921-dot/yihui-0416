內部簽核 + 庫存領用｜需求規格

Why

我想解決的問題

我用「規格」做成可交付成果：把需求拆成流程、資料、權限、例外與驗收標準，降低返工、提升溝通效率。

痛點

  • 權責不清：誰核准、誰領用、誰出庫不一致
  • 庫存不準：盤點才發現缺料/多料
  • 追溯困難：找不到領用原因與使用者

目標

  • 簽核流程確保權責清楚
  • 庫存異動確保資料一致
  • 報表讓管理可視化
開啟完整規格（md） 開啟流程 Demo

可搭配作品

  • ERD + SQLite 報表作品
  • SQL 採購/庫存報表作品

Flow

核心流程（可視覺化）

用狀態流轉把流程定義清楚，方便驗收。

flowchart TD
  A[建立申請] --> B[PENDING_APPROVAL]
  B --> C{主管決策}
  C -- 退回 --> D[REJECTED\n填寫原因] --> A
  C -- 核准 --> E[APPROVED]
  E --> F{倉管出庫}
  F -- 庫存不足 --> G[阻擋出庫\n提示補貨/調整]
  F -- 成功 --> H[FULFILLED\n寫入庫存異動]

Acceptance

驗收與測試（摘要）

我在規格中把「什麼叫做完成」寫清楚，避免做完但不可用。

驗收標準

  • 核准/退回結果能回寫到申請人列表
  • 只有已核准的申請可出庫，且會正確扣庫存
  • 每筆出庫都可追溯到申請單、簽核與出庫人

測試清單

  • 庫存不足/超量/品項停用等例外
  • 非本部門主管不可核准
  • 重複出庫不可（避免扣兩次）
