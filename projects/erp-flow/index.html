採購到入庫（P2P）

Goal

我想展示什麼能力

把 ERP 流程拆成「角色/狀態/資料」，並能被驗收；同時能從流程推導出管理報表指標。

流程目標

  • 讓採購需求可追溯、可核准、可入庫
  • 讓「誰提出、誰核准、誰收貨」清楚分工
  • 讓 PR → PO → GRN 資料一致串起來

角色與權限（簡化）

  • 請購人：建立/送出/取消（未核准前）
  • 主管：核准/退回
  • 採購：建立 PO、選供應商、議價
  • 倉管：收貨、驗收、入庫

作品原始文件

  • `flow.md`（完整整理）
  • 搭配：需求規格（簽核/庫存）

Flow

流程圖（可直接展示）

Mermaid 直接渲染，讓流程在網站上就是成品。

flowchart TD
  A[請購人：建立請購單 PR] --> B[送出 PR]
  B --> C{主管審核}
  C -- 退回 --> D[PR 狀態：REJECTED\n填寫原因] --> A
  C -- 核准 --> E[PR 狀態：APPROVED]
  E --> F[採購：建立採購單 PO\n選供應商/價格/交期]
  F --> G[送出 PO]
  G --> H{到貨？}
  H -- 否 --> I[等待/追蹤交期]
  I --> H
  H -- 是 --> J[倉管：收貨 GRN]
  J --> K{驗收結果}
  K -- 不合格 --> L[退貨/換貨\n建立退貨單]
  K -- 合格 --> M[入庫：Stock IN\n更新庫存]
  M --> N[完成：可對帳/可報表]

Deliverables

單據與狀態（可驗收）

把流程落地時最怕「狀態亂跳」。我用狀態機方式整理。

PR（請購單）

  • 狀態：DRAFT → PENDING_APPROVAL → APPROVED / REJECTED / CANCELLED
  • 欄位：請購人/部門/需求日/用途/明細（品項、數量、預算）

PO（採購單）

  • 狀態：DRAFT → SENT → PARTIALLY_RECEIVED / RECEIVED / CANCELLED
  • 欄位：供應商/交期/單價/付款條件/明細

GRN（收貨單）

  • 狀態：CREATED → QC_PASSED / QC_FAILED
  • 欄位：到貨數量/驗收結果/短溢到貨原因/倉別
